 1. 第 1 步：   生成密钥对
打开 PowerShell (右键以管理员身份运行，虽非必须但推荐)，输入以下命令：

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```
说明：

* -t ed25519：使用更安全的加密算法（如果服务器太老不支持，可改为 -t rsa -b 4096）。

* -C：备注信息，通常是你的邮箱，方便识别。

* 交互提示：

  1. Enter file in which to save the key: 直接按 回车 (使用默认路径 C:\Users\你的用户名\.ssh\id_ed25519)。

  2. Enter passphrase: 直接按回车 (留空)。

  * 注意：为了达到“完全免密”自动化的效果，这里必须留空。如果设置了密码，每次使用密钥时还需要输入一次 passphrase。

  3. Enter same passphrase again: 再次按 回车。
   
2. 第 2 步：将公钥上传到服务器
Windows 自带的 ssh-copy-id 命令可能不可用（取决于版本），我们可以用 type + ssh 的组合命令来手动完成。

在 PowerShell 中执行（替换为你的实际信息）：
```powershell
#语法：type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh 用户名@服务器IP "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"

type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh root@192.168.1.100 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```
* 解释：

  * type ...: 读取本地公钥内容。

  * |: 管道符，将内容传给后面的命令。

  * ssh root@...: 登录服务器。

  * mkdir -p ~/.ssh: 确保服务器用户目录下有 .ssh 文件夹。

  * cat >> ~/.ssh/authorized_keys: 将公钥内容追加到授权文件中。

* 操作：此时会提示你输入一次服务器的登录密码，输入后回车。

3. 第 3 步：验证登录
再次尝试连接：
```
ssh root@192.168.1.100
```

---------------------------------------------------------------------

如果直接进入了命令行提示符（如 root@ubuntu:~#），没有提示输入密码，则配置成功！

Bad owner or permissions 报错

1. 如果你是在 Windows (PowerShell/CMD) 上
Windows 的权限系统与 Linux 不同，但 OpenSSH 客户端会检查 ACL（访问控制列表）。最简单的修复方法是重置该文件的权限，只保留当前用户和系统管理员的权限。

请在 PowerShell (管理员身份) 中运行以下命令（替换为你的实际用户名和路径）：
```powershell
#定义变量 (通常路径是 C:\Users\你的用户名\.ssh\id_ed25519)
$KeyPath = "$env:USERPROFILE\.ssh\id_ed25519"
$PubKeyPath = "$env:USERPROFILE\.ssh\id_ed25519.pub"
$ConfigPath = "$env:USERPROFILE\.ssh\config" # 如果有这个文件也要修

#1. 清除所有现有权限
icacls $KeyPath /reset
icacls $PubKeyPath /reset
if (Test-Path $ConfigPath) { icacls $ConfigPath /reset }

#2. 只赋予当前用户 (你) 和 管理员组 完全控制权
#%USERNAME% 会自动替换为你当前的登录名
icacls $KeyPath /grant:r "$($env:USERNAME):(F)"
icacls $PubKeyPath /grant:r "$($env:USERNAME):(F)"
if (Test-Path $ConfigPath) { icacls $ConfigPath /grant:r "$($env:USERNAME):(F)" }

#3. 移除继承权限 (确保没有奇怪的父级权限混入)
icacls $KeyPath /inheritance:r
icacls $PubKeyPath /inheritance:r
if (Test-Path $ConfigPath) { icacls $ConfigPath /inheritance:r }

Write-Host "✅ 权限已修复！请再次尝试 ssh 连接。"
```
手动图形界面做法 (如果不喜欢用命令)：

1. 进入 C:\Users\你的用户名\.ssh\ 文件夹。

2. 右键点击 id_ed25519 (私钥，没有后缀) -> 属性 -> 安全。

3. 点击 高级 -> 禁用继承 -> “删除所有已继承的权限”。

4. 确保列表中只有你的当前用户账号 和 Administrators (或 SYSTEM)。

5. 如果有 Users、Everyone 或其他陌生账号，全部删除。

6. 确保你的账号拥有 完全控制 (Full Control)。

7. 对 id_ed25519.pub 和 config (如果有) 做同样操作。


-------------------------------------------------------------------------
2. 如果你是在 Mac 或 Linux 上
直接在终端运行以下命令：
```bash
#修复私钥权限 (必须是 600，即只有所有者可读写)
chmod 600 ~/.ssh/id_ed25519

#修复公钥权限 (通常是 644，所有者读写，其他人只读)
chmod 644 ~/.ssh/id_ed25519.pub

#修复 .ssh 目录权限 (必须是 700，即只有所有者可进入)
chmod 700 ~/.ssh

#如果有 config 文件，也需修复
chmod 600 ~/.ssh/config
```
