 清理 Docker 日志主要分为两个步骤：紧急清理现有日志以释放磁盘空间，以及配置日志策略以防止问题再次发生。

🚨 紧急清理：快速释放磁盘空间
当磁盘空间告急时，你可以使用以下命令快速清空一个或所有容器的日志文件。此操作无需停止容器，对服务影响最小。

### 清理单个容器的日志
如果你只想清理特定容器的日志，可以使用这条组合命令，它会自动找到该容器的日志文件路径并清空：
```
truncate -s 0 $(docker inspect --format='{{.LogPath}}' <容器名称或ID>)
```

清理所有容器的日志
如果你想一次性清空所有容器的日志，可以使用通配符命令：
```
sudo truncate -s 0 /var/lib/docker/containers/*/*-json.log
```
### 重要提示
使用 truncate 而非 rm：truncate -s 0 命令会将文件内容清空，但保留文件本身。这可以确保正在写入日志的 Docker 进程不会出错。千万不要使用 rm 命令直接删除日志文件，否则可能导致容器异常或磁盘空间无法真正释放。

需要 root 权限：执行上述命令通常需要 sudo 权限。

### 🛡️长期方案：配置日志大小限制
为了避免日志文件无限增长，最佳实践是配置 Docker 的日志轮转策略。这样，当日志文件达到指定大小后，Docker 会自动进行归档和清理。

修改全局配置（推荐）
这是最推荐的方法，会对之后创建的所有容器生效。

1. 编辑或创建 Docker 守护进程的配置文件 /etc/docker/daemon.json。

2. 添加或修改以下内容：
```
{
  "log-driver": "json-file",
 "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```
* max-size: 单个日志文件的最大大小，例如 10m 表示 10MB。
* max-file: 最多保留的日志文件数量。

3. 保存文件后，重新加载 Docker 配置使更改生效：
```
sudo systemctl reload docker
```

注意：建议使用 reload 而不是 restart，这样可以避免重启所有正在运行的容器，保证业务连续性。
